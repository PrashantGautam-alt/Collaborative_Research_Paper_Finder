DEPLOY TO GITHUB + STREAMLIT CLOUD
====================================


YOUR REPO
---------
https://github.com/PrashantGautam-alt/Collaborative_Research_Paper_Finder

The repo already exists. You are pushing updated files to it.


STEP 1 — STAGE ALL FILES
--------------------------
Run in terminal:

    cd "/Users/prashantgautam/Documents/TEST FILES/python/Collaborative_Research_Paper_Finder"
    git add .

This stages all new and modified files. The venv/, .env, and .claude/
folders are already in .gitignore and will not be included.


STEP 2 — COMMIT
----------------
Run:

    git commit -m "Rebuild: 5-agent pipeline with Streamlit UI"

You will see a list of files included in the commit.


STEP 3 — PUSH TO GITHUB
------------------------
Run:

    git push origin main

After it completes, open your repo on GitHub and confirm these are visible:
  - app.py
  - pipeline.py
  - agents/
  - utils/
  - archive/
  - .env.example
  - requirements.txt
  - README.md

If git asks for a password, GitHub no longer accepts your account password.
You need a Personal Access Token (PAT). To generate one:

  1. github.com -> profile picture -> Settings
  2. Scroll to the bottom of the left sidebar -> Developer settings
  3. Personal access tokens -> Tokens (classic)
  4. Generate new token (classic)
  5. Name: anything (e.g. "my laptop")
  6. Expiration: 90 days
  7. Check the "repo" box
  8. Click Generate token
  9. Copy it immediately — it is only shown once

When git prompts for your password, paste the token.


STEP 4 — DEPLOY ON STREAMLIT CLOUD
------------------------------------
4a. Go to https://share.streamlit.io
    Sign in with your GitHub account.

4b. Click "New app".

4c. Fill in the form:
      Repository : PrashantGautam-alt/Collaborative_Research_Paper_Finder
      Branch     : main
      Main file  : app.py

4d. Click "Advanced settings" -> "Secrets" tab.
    Paste both keys exactly like this:

      GEMINI_API_KEY = "your_gemini_key_here"
      SEMANTIC_SCHOLAR_API_KEY = "your_semantic_scholar_key_here"

    Both keys are required. The quotes around the values are required.
    Get your Gemini key at:            https://aistudio.google.com/app/apikey
    Get your Semantic Scholar key at:  https://www.semanticscholar.org/product/api

    Why secrets and not .env:
    Your .env file is gitignored and never reaches GitHub. Streamlit Cloud
    reads these secrets instead and injects them as environment variables at
    runtime. Your code reads them with os.environ.get() — identical to how
    it reads .env locally.

4e. Click Deploy.
    Streamlit Cloud installs requirements.txt and starts the app.
    This takes 2-4 minutes. You will see a live build log.

4f. Once the build finishes you get a URL:
      https://prashantgautam-alt-collaborativeresearch.streamlit.app

    Open it, type a query, click Find Papers.
    If a report appears, deployment is complete.
    Save that URL — it goes on your resume.


COMMON ERRORS
-------------
ModuleNotFoundError: No module named 'google.genai'
  requirements.txt must have: google-genai
  (not google.genai, not google-generativeai)
  Fix: correct requirements.txt, push again, then Reboot app on Streamlit Cloud.

ModuleNotFoundError: No module named 'dotenv'
  requirements.txt must have: python-dotenv
  Fix: same as above.

API calls fail or return quota errors
  Check Settings -> Secrets on Streamlit Cloud.
  Must look exactly like:
    GEMINI_API_KEY = "your-key-here"
    SEMANTIC_SCHOLAR_API_KEY = "your-key-here"
  No extra spaces. Quotes around both values. Save and reboot.

Build log says "No such file: app.py"
  The Main file field must be set to: app.py
  app.py must be at the root of the repo, not inside a subfolder.
